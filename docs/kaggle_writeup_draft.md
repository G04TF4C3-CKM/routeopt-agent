# Kaggle Write‑up Draft (Capstone)

**Title**: RouteOpt Agent – Agent‑Assisted Vehicle‑Routing Optimization

**Subtitle**: Leveraging a legacy VRP solver behind a clean, agent‑style workflow.

**Track**: Agents for Business (Kaggle Vibe Coding)

---

### Problem Statement

Dispatch centers must assign drivers to a set of loads while respecting a maximum driver‑time limit.  The combinatorial nature of the vehicle‑routing problem makes manual planning error‑prone and inefficient.  A compact, deterministic optimizer can dramatically reduce the number of drivers needed and lower total route time, delivering cost savings.

### Solution Overview

* **Backend** – A custom residual‑graph VRP optimizer (`src/legacy_vrp.py`) wrapped by `src.routing_engine.solve_routing_problem`.  The optimizer starts with one driver per load and repeatedly merges routes while staying under the time limit.
* **Workflow** – An explicit agent‑style pipeline (`src.agent_workflow.run_routing_workflow`) that loads a scenario, validates it, runs the optimizer, analyses the numeric result, and produces a natural‑language explanation.
* **Front‑end** – A minimal Streamlit UI (`streamlit_app.py`) that lets a user pick a bundled scenario file, set the driver time limit, execute the workflow, and view the key metrics and a simple route‑length chart.

The UI is deliberately lightweight (no heavy CSS, only Streamlit defaults) while exposing the full set of backend results:
- Initial / final driver count
- Initial / final total route time
- Driver‑time savings and feasibility
- Final routes per driver
- A concise explanation generated by `result_analyzer.explain_solution`

### Architecture

```
User / UI / CLI
    |
    v
RouteOpt Agent Workflow
    |
    +--> LoadScenarioNode
    +--> ValidateScenarioNode
    +--> RunOptimizerNode
    +--> AnalyzeSolutionNode
    +--> ExplainSolutionNode
    |
    v
Stable routing_engine.solve_routing_problem()
    |
    v
legacy_vrp.py custom residual‑graph VRP optimizer
```

The Streamlit app is just another entry point that calls the same deterministic workflow, guaranteeing that the CLI and UI produce identical results.

### Implementation Details

* **Scenario files** – Stored under `data/*.txt`; the UI presents them via a dropdown selector.
* **Result handling** – The workflow returns a `WorkflowState` containing `result`, `analysis`, and `explanation`.  The Streamlit app reads these fields directly; no additional processing is performed.
* **Visualization** – A Matplotlib bar chart visualises the number of stops per driver, providing a quick visual sanity‑check.
* **Safety** – All file reads are guarded by `safety.assert_local_readable_file`.  No network calls, secrets, or external services are used.

### Safety & Security

- Only local files under the repository may be read.
- The app runs entirely offline; no API keys or credentials are required.
- Input validation ensures malformed scenario files raise a clear error displayed in the UI.
